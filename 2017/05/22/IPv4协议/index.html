IPv4协议

发表于 2017-05-22

IPv4(Internet Protocol Version 4)

概述

  • IPv4协议是TCP/IP协议的核心，所有TCP、UDP、IMCP等协议都是以IP数据报文格式传输；
  • IPv4协议是一种无连接、不可靠的协议，IPv4运行与分组交换的链路层(如：以太网)之上；
  • IPv4不保证分组数据能够被送达目的地，也不能保证所有分组数据能够按照正确的顺序无重复的送达目的地，只会尽最大努力发送分组数据，可靠性与时序性需要由上层传输协议(如：TCP协议)来控制；

IPv4协议头

IPv4报文结构图


IPv4报文包含一个首部和一份数据

IPv4首部包含14个字段，其中13个是必须的，第14个是可选的（选项字段），首部字段以网络字节序(大端序)封装

  • Version（版本）

    • IP报文首部的第一个字段是4 bits版本字段，对于IPv4协议来说，该字段的值为4；

  • Header Length（首部长度）

    • 第二个字段是4 bits的首部长度；
    • 该字段说明首部有多少个Bytes的长度；
    • IPv4首部包含有选项字段，所以首部大小不是固定的Header Length也是用来确定实际数据的偏移量的。

  • Service（服务）

    • 暂时不知道什么意思

  • Explicit Congestion Notification（显示拥塞通知）

    • 在不丢球报文的同时通知发送端网络拥塞；
    • IP层的ECN是可选功能，发送端与接收端都必须支持，并且协商好同时使用；
    • IP层的ECN可以与TCP协议层的ECN互相参考；

  • Total Length（报文长度）

    • Total Length定义了IPv4的报文长度，包含首部与实际数据，以字节为单位；
    • 该字段最小值为20 Bytes（20 Bytes首部长度 + 0 Bytes数据长度），最大值为2^16 - 1 = 65535；
    • IPv4协议规定所有主机必须支持最小576 Bytes的报文（上层协议数据512 Bytes + IP首部最大字节数60 Bytes + 4 Bytes富余量）；

  • Identification（标识符）

    • 该字段被用来唯一的标识一个报文的所有分片；
    • 每产生一个数据报文，会为已有计数器加1，并赋值给该字段；

  • Flags（标志）

    • 该字段的3个bits用于控制和识别分片；
    • Reserved bit是一个保留位，值必须为0；
    • Don’t fragment是否禁止分片标志位，当其值为0时时允许分片；
    • More fragments时更多分片标志位，其值为1表示后面还有分片，其值为0时表示已经是最后一个分片；

  • Fragment Offset（分片位移）

    • 该字段指明每个分片相对于原始报文开头的偏移量，以8 Bytes为单位；

  • Time To Live（TTL，存活时间）

    • 该字段的作用数避免数据报文在网络中永久存在；
    • TTL是以秒为单位；
    • 数据报文每经过一个路由器都会将该字段值减一，当该字段值等于0的时候，报文不再向下一跳传送，并且会被当前路由器丢弃；
    • 该字段最大值为255，一般会将该字段值设为64或32；

  • Protocol（协议）

    • 该字段定义了报文的实际数据使用的协议（如第一张图中使用的UDP协议）;

  • Header Checksum（首部校验和）

    • 该校验和只会针对IP首部进行校验，不会包括实际数据部分；
    • 在每一跳的路由器中，都需要对报文进行首部校验和的重新计算，并与该字段进行比较，如果不一致，则此数据报文将会被路由器丢弃；
    • 重新计算的必要性是因为在每一跳的时候都会对IP首部字段（如：TTL, Flag, Offset等）进行修改；
    • 不对实际数据进行校验是出于性能考虑，数据的校验是有上层协议进行校验处理，如UDP与TCP协议都有校验和字段；

  • Source（源IP地址）

  • Destination（目标地址）

  • Options（选项）

    • EOL标识选项列表结束；

IP的分片与组装

不同的数据链路层不仅仅在传输速度上存在差异，并且在帧结构与帧尺寸上也不尽相同，为了使不同的网络能够顺利的交换数据报文，IP协议必须独立于链路层传输，在这样的情况下，就必须实现一种能够自动调整传输单元大小的机制来适应不同的链路层传输技术，于是就产生了IP数据报文的分片与组装。

分片

–

  • 当一个路由设备收到一个IP数据报文后，会将报文的长度与该设备的MTU进行比较，如果数据报文长度比该设备的MTU大，那么该设备必须对数据报文进行切片处理；
  • 设备将数据报文分成数片，每一片的长度必须小于等于该设备的MTU减去IP首部长度；
  • 被分片的数据会被存放到独立的IP报文中，并且会对这些独立的数据报文进行如下的修改：
    • 报文首部总长度字段（Total Length）会被设置为此分片的长度；
    • 更多分片标志（More Fragments）标志会被设置为1，如果是最后一个分片则为0；
    • 分片的偏移量字段（Fragment Offset）会被调整为合适的值；
    • 每个分片的首部校验和会被重新计算；
  • 如果数据报文经过路径的MTU减小了，那么已经被分片的数据报文会被再次分片；

组装

–

  • 当接收端发现IP数据报文的以下项目发生变化，便知道该数据报文被分片：
    • 更多分片标志被设置为1；
    • 分片偏移量字段值不为0；
  • 一旦接收端收齐了所有分片，便将所有分片按照正确的顺序组装起来，发送给上层协议栈处理；

IPv4地址

  • IPv4为网络每个节点分配全球唯一的IP地址；
  • IPv4使用32bits整数来表达一个地址，该地址最大范围是2^32大约40亿；

注：随着IPv4地址不断的被分配给用户，地址枯竭的问题也随之产生。为了延缓地址枯竭的速度，随之产生了分类网络、无类别域间路由、网络地址转换等技术。

分类网络(Classful Addressing)

分类网络是在1993年左右用于描述网络体系的一个术语，它将IPv4地址空间分为五个类别，每个类别地址都由它们的前3位来标识；

地址类别 地址范围 网络ID/主机ID(N：网络，H：主机) 默认子网掩码 网络数量 网络可用地址
A 1 - 126 N.H.H.H 255.0.0.0 126(2^7 - 2) 16,777,214 (2^24 – 2)
B 128 - 191 N.N.H.H 255.255.0.0 16,382 (2^14 – 2) 65,534 (2^16 – 2)
C 192 - 223 N.N.N.H 255.255.255.0 2,097,150 (2^21 – 2) 254 (2^8 – 2)
D 224 - 239 多播预留
E 240 - 254 实验性的，用于研究

按位来表示

  • n 表示该二进制位是网络位
  • H 表示该二进制位是主机位
  • X 表示该二进制位无特定作用

注：A类IP地址中的127.0.0.0 至 127.255.255.255 不能被使用, 它们预留用于环回地址和诊断功能

无类别域间路由(Classless Inter-Domain Routing, CIDR)

  • 无类别域间路由是一个用于给用户分配IP地址以及在互联网上有效的路由IP数据包的对IP地址进行归类的方法。
  • CIDR主要是一个按位的、基于前缀的，用于解释IP地址的标准；

应对IPv4地址枯竭

随着互联网的增长，出现了各类技术来应对IPv4地址的耗尽，包括：

  • 网络地址转换(NAT)
  • 专用网络技术
  • 动态主机设置协议(DHCP)
  • 基于名字的虚拟主机
