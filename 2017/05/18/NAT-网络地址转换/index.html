网络地址转换(NAT)

​网络地址转换(Network Address Translation, NAT)，是一种在数据包通过路由器或者防火墙等NAT网关时，重写IP协议头中的Source IP Address或者Destination IP Address的技术。这种技术的使用场景为：多台主机只能通过一个公网IP来访问Internet Network。

内部专用网络

  • 在IPV4的地址空间中，有三个地址块被保留为专用网络
    • 这些地址不能再专用网络之外路由；
    • 专用网络内的主机不能直接与公共网络通信，必须通过网络地址转换(NAT)才可以做到与公共网络通信；
类别描述 私有网络 子网掩码 地址范围
一个A类 10.0.0.0 255.0.0.0 10.0.0.0 - 10.255.255.255
连续的16个B类 172.16.0.0 - 172.31.0.0 255.240.0.0 172.16.0.0 - 172.31.255.255
连续的256个C类 192.168.0.0 255.255.0.0 192.168.0.0 - 192.168.255.255

NAT网关处理数据报文

  • 网络被分为私有网络和公网两个部分，NAT网关设置在私有网络的出口位置，双向流量必须都要经过NAT网关；
  • 网络访问只能由私有网络一端发起，公网无法主动访问私有网络内的主机；
  • NAT网关在两个访问方向上完成两次地址的翻译或转换，在出口方向做源信息替换，在入口方向做目的信息替换；
  • NAT网关对通信双方保持透明；
  • NAT网关为了实现双向转换或翻译，需要维护一张关联表，把会话信息保留下来；

NAT类型

完全圆锥型NAT(Full Cone NAT)

  • 一旦一个内部地址(iAddr:port1)映射到外部地址(eAddr:prot2)，所有发自iAddr:port1的包都经由eAddr:port2向外发送；
  • 任意外部主机都能通过给eAddr:port2给iAddr:port1发送数据包；

受限圆锥型NAT(Address-Restricted Cone NAT)

  • 一旦一个内部地址(iAddr:port1)映射到外部地址(eAddr:port2)，所有发送自iAddr:port1的数据包都经由eAddr:prot2向外发送；
  • 任意外部主机(hostAddr:any)都能通过eAddr:port2向iAddr:port1发送数据包，但是数据包能够到达iAddr:prot1的前提是：iAddr:port1之前向hostAddr:any发送过数据包，并且数据包到达了hostAddr:any。”any”是指端口不受限制；

端口受限圆锥型NAT(Port-Restricted Cone NAT)

  • 一旦一个内部地址(iAddr:port1)映射到外部地址(eAddr:port2)，所有发自iAddr:port1的数据包都经由eAddr:port2向外发送；
  • 一个外部主机(hostAddr:port3)能够通过eAddr:port2向iAddr:port1发送数据包，但是数据包能够到达iAddr:port1的前提是：iAddr:port1之前已经向hostAddr:port3发送过数据包，并且数据包到达了hostAddr:port3；

对称NAT(Symmetric NAT)

  • 每一个来自相同内部IP与端口，到一个特定目的地址和端口的请求，都将映射到一个独特的外部IP地址和端口；
  • 同一个内部IP和端口发送到不同的目的地址和端口的数据包，都使用不同的映射；
  • 只有曾经收到过内部主机数据包的外部主机，才能够把数据包发回；
